WebHID Settings

Daemon as NM host

Talk to the daemon directly over native messaging (skip the webhid-native-messaging forwarder and Unix socket). Requires the webhid.daemon_nm_host native-messaging manifest to be installed and the daemon to run as your user (with the udev rule). Lower latency, but no root-daemon fallback. The addon reconnects automatically when toggled.

Control Plane

Native Messaging: enumerate/open/close via NM host (default).
WebSocket: after NM handshake (control token), route enumerate/close via WS text frames. Lower latency for repeated control operations.

Data Plane

WebSocket: spawn a Web Worker that owns the WS connection to the daemon. Input reports are forwarded to the page via postMessage with ArrayBuffer transfer (off-main-thread, zero-copy). No SAB, no COOP/COEP.
Native Messaging: route all data (sendReport, input reports, feature reports) through the NM host via the content-script bridge. No WS. Fallback when WS is blocked.

Fire-and-forget sendReport

Resolve sendReport Promise immediately without waiting for daemon ack (faster, but no error reporting on write failure)

Device Picker Mode

Modal: show a shadow DOM overlay on the page when requestDevice() is called (default, matches native WebHID behavior).
Page Action: switch the url bar icon to alert mode and open the device picker as a popup. User must click the icon to open the picker. Firefox only — Chromium will fall back to modal.

Log Level

Controls console output verbosity. Error = errors only, Warn = +warnings, Info = +info, Debug = +debug (includes performance timing). Reload pages after changing.
